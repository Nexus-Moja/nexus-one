# NEXUS ONE Shared Experience Standard

## Book a Ride

Every public **Book a Ride** control now uses the same wording, visual treatment, keyboard behavior, and booking workflow.

- Homepage navigation: opens the booking dialog.
- Homepage hero: opens the booking dialog.
- Hero carousel: opens the booking dialog with the featured service preselected.
- Service cards: open the same booking dialog with the selected service preselected.
- Homepage CTA: opens the same booking dialog.
- LiveCare and Operations: link to `/?book=1`, which automatically opens the same homepage booking dialog.
- Optional service deep links may use `/?book=1&service=<service name>`.

## Shared module hero

The visual language from the homepage persists across operational modules:

- Nexus logo and white navigation bar
- Deep navy-to-blue gradient
- Cyan ambient glow
- Uppercase Nexus product eyebrow
- Large, concise headline
- Supporting explanatory copy
- Red gradient Book a Ride CTA
- Secondary module-specific action

LiveCare and Operations use this shared hierarchy while preserving the purpose of each module.

## Accessibility

The shared controls retain:

- Visible focus indicators
- Minimum target sizes
- Semantic buttons and links
- Reduced-motion behavior
- High-contrast compatibility
- Descriptive navigation labels
